## 🎯 1: Проблема
**Изучение языка через аудио и текст сегодня разорвано или слишком фрагментировано:**
- 📖 Чтение с переводом не развивает аудирование и восприятие речи на слух.
- 🎧 Обычные аудиокниги на иностранном слишком сложны для среднего уровня без текстового контекста.
- ⏳ Ручная конвертация текста в аудио с последующим монтажом переводов занимает часы.
- 🧩 Существующие сервисы либо работают только с готовой библиотекой, либо не предлагают структуру *[оригинал → перевод → пояснение]* в одном аудиопотоке.

---

## 💡 2: Решение (УТП)
**AudioLingua — сервис, который превращает любой загруженный текст в структурированную аудиокнигу с обучающей логикой:**
1. 📥 Загрузка текста (глава книги, статья, конспект).
2. ⚙️ Настройка размера "чанка" (например, читать по 3 предложения подряд).
3. 🤖 Автоматическая серверная обработка:
   - `[N предложений на EN]` → синтез речи.
   - `[Перевод на RU]` → синтез речи.
   - `[Пояснение идиом/грамматики]` → текстовая или аудио-вставка.
4. 🔗 **Серверная склейка:** все фрагменты бесшовно объединяются в один `chapter.mp3` через FFmpeg.
5. 📚 Возможность объединить несколько сгенерированных глав в единую `audiobook.mp3`.

> ✨ **Уникальность**: Первый сервис, сочетающий `загрузку своего текста` + `настраиваемую длину аудио-сегментов` + `автоматическую компиляцию в готовую аудиокнигу`.

## ⚙️ 5: MVP Функционал и Стек (Актуальная реализация)

### ✅ Что уже работает в текущей версии
- **Умная сегментация:** Разбиение текста на предложения и группировка их в настраиваемые чанки (1–10 предложений).
- **Реальный бэкенд:** Полноценный API на FastAPI, использующий Yandex Cloud для перевода и синтеза речи (обход ограничений через `curl`).
- **Аудио-конкатенация:** Автоматическая склейка аудиофайлов на сервере через FFmpeg (без потери качества и перекодирования), включая финальную сборку главы и потенциальную сборку книги.
- **Стабильный UX:** Плавные CSS-анимации переходов между экранами (устранено "дёргание" интерфейса), корректная работа управления скоростью воспроизведения (`playbackRate`).
- **Экспорт:** Прямое скачивание скомпилированного MP3-файла главы одной кнопкой.

### 🧩 Технологический стек
| Слой | Решение | Почему выбрано |
|------|---------|----------------|
| **Backend** | Python, FastAPI, Uvicorn | Высокая скорость, нативная поддержка `async`, простота интеграции с OS-утилитами (FFmpeg, curl). |
| **API** | Yandex Translate, Yandex SpeechKit | Лучшее качество синтеза русской и английской речи, поддержка папок и API-ключей. |
| **Аудио-обработка** | FFmpeg (`concat demuxer`) | Мгновенная склейка MP3 без перекодирования (lossless), экономия ресурсов сервера. |
| **Frontend** | HTML5, CSS3, Vanilla JS (ES6+) | Нулевая зависимость от тяжёлых фреймворков (React/Vue), мгновенная загрузка, прямой контроль над DOM и `<audio>` API. |
| **Хранение** | Локальная ФС (папка `output/`) | Для MVP: простота отладки. В продакшене легко заменяется на S3 (Yandex Object Storage). |
